# PR Guidelines

## PR Types

Every PR title must be prefixed with a type. This enables filtering git history by intent.

| Prefix | Meaning | Review expectation |
|--------|---------|-------------------|
| `feat:` | Production-ready feature | Full review, tests required |
| `fix:` | Bug fix | Focused review on the fix |
| `proto:` | Fast prototyping / proof of concept | Minimal review, rough edges expected, follow-up cleanup PR needed |
| `arch:` | Architecture shift / refactor | Deep review, no behavior change |
| `infra:` | CI/CD, tooling, dev environment | Light review |
| `docs:` | Documentation only | Skim |

PRs must pass CI (green checks) before merge — no exceptions including `proto:`.

## PR Requirements

Every PR that changes architecture MUST include:
1. **Before/After dependency graph** — generated by `bun run docs:deps` (dep-cruiser SVG from real imports). Attach SVG from main (before) and branch (after) to PR.
2. **CLAUDE.md update** — if packages added/removed/renamed
3. **Definition of Done checklist** — all items checked before declaring ready

## PR Review Process

After pushing a PR:
1. **Wait for CI** — all checks must pass before requesting review
2. **Request review** — trigger automated review (Copilot) or ask human
3. **Read every review comment** — do not skip any
4. **Reply to each comment** on GitHub with what was done (fix, or why not applicable)
5. **Fix all valid issues** — commit and push fixes
6. **Re-request review** if changes were made
7. **CI must be green** — if build fails, fix it before merge

## Committing with Git

Only create commits when requested by the user. When the user asks you to create a new git commit, follow these steps:

1. Run git status, diff, and log commands in parallel
2. Analyze changes and draft a concise commit message
3. Add relevant untracked files and create commit with:
   ```
   🤖 Generated with [Claude Code](https://claude.com/claude-code)

   Co-Authored-By: Claude <noreply@anthropic.com>
   ```
4. Never push unless explicitly requested
5. Never skip hooks or force push without explicit request
